Data Definition Language

1 Introduction

This document is the specification of the Data Definition Language. Programs of this language are sets of UTF8 files of this language and describe structured data for the purpose of for exchanging that data between entities (humans and machines alike). The language provides scalar types (boolean type, number type, string type, and void type) as well as aggregate types (map values and list values).

The language is insipired by JSON (see ECMA-404 The JSON interchange syntax, 2nd edition, December 2017 for more information). A conversion between JSON and DDL is possible without the loss of data.

3 Syntactical Specification

The syntactical grammar describes the translation of the words into sentences. The goal non-terminal of the syntactical grammar is the sentence syn:Sentence symbol.

The words lex:Whitespace and lex:LineTerminator are removed from the sequence of words before the translation to sentences is performed.

The goal sentence syn:Sentence is defined by

syn:Sentence : syn:Value

The sentence syn:Value is defined by

syn:Value : syn:Map
syn:Value : syn:List
syn:Value : syn:String
syn:Value : syn:Number
syn:Value : syn:Boolean
syn:Value : sny:Void

The sentence syn:String is defined by

syn:String : lex:String

The sentence syn:Number is defined by

syn:Number : lex:Number

The sentence syn:Boolean is defined by

syn:Boolean : lex:Boolean

The sentence syn:Void is defined by

syn:Void : lex:Void

The sentence syn:Map is defined by

syn:Map : lex:LeftCurlyBracket
      syn:MapBody
      lex:RightCurlyBracket

syn:MapBody : syn:MapBodyElement syn:MapBodyRest
syn:MapBody : ε

syn:MapBodyRest : lex:Comma syn:MapBodyElement syn:MapBodyRest
syn:MapBodyRest : lex:Comma
syn:MapBodyRest : ε
syn:MapBodyElement : lex:Name lex:Colon syn:Value

The sentence syn:List is defined by

syn:List : lex:LeftSquareBracket
       syn:ListBody
       lex:RightSquareBracket

syn:ListBody : syn:ListBodyElement syn:ListBodyRest
syn:ListBody : ε

syn:ListBodyRest : lex:Comma

4 Semantical Specification

The Data Definition Language language describes structured data in terms of typed values. That is, each sentence of the syntactical specification translates to values each of which has type. The Data Definition Language knows six basic types \(\textit{List}\) and \(\textit{Map}\), which are the so called aggregate types, and \(\textit{Boolean}\), \(\textit{Number}\), \(\textit{String}\), and \(\textit{Void}\), which are the so called scalar types. The type \(\textit{Value}\) is defined as the union \[\begin{aligned} \textit{Value} =&\;\textit{List}\\ \cup&\;\textit{Map}\\ \cup&\;\textit{Boolean}\\ \cup&\;\textit{Number}\\ \cup&\;\textit{String}\\ \cup&\;\textit{Void} \end{aligned}\] There exists type \(\textit{Name}\) representing .

To define the semantics of the language, a translation function \[ \sigma : \left(\textit{Terminals} \cup \textit{NonTerminals}\right)* \rightarrow \textit{Name} \cup \textit{Value} \] is defined which translates from the set of strings generated by the grammar to types.

4.1 Boolean type

The type \(\textit{Boolean}\) type has two values \(\textit{true}\) and \(\textit{false}\) which are expressed in the language by the words \(\texttt{lex:true}\) and \(\texttt{lex:false}\), respectively (as defined in the syntactical grammar).

\[\begin{aligned} \sigma\left(\texttt{syn:Boolean}\right) &= \sigma\left(\texttt{lex:Boolean}\right)\\ \sigma\left(\texttt{lex:false}\right) &= \textit{false}\\ \sigma\left(\texttt{lex:true}\right) &= \textit{true} \end{aligned}\]

4.2 List type

The type \(\textit{List}\) represents list of elements. Each element is a DDL node of type \(\textit{Value}\). A DDL node of type \(\textit{List}\) is expressed in the language by the sentence \(\texttt{syn:List}\) (as defined in the syntactical grammar).

The semantical representation is defined as \[\begin{aligned} &\sigma\left(\texttt{syn:List}\right)\\ =&\sigma\left(\texttt{lex:LeftSquareBracket}\; \texttt{syn:ListBody}\; \texttt{lex:RightSquareBracket} \right)\\ =&[ \sigma\left(\texttt{syn:ListBody} \right) ]\\ =&[ \sigma\left(\texttt{syn:ListBodyElement}\; \texttt{syn:ListBodyRest} \right) ]\\ =&[ \sigma\left(\texttt{syn:ListBodyElement} \right) ] \circ \sigma\left(\texttt{syn:ListBodyRest} \right) \end{aligned}\] with \[ \sigma\left(\texttt{syn:ListBodyElement} \right) = \sigma\left(\texttt{syn:Value} \right) \] and \[\begin{aligned} &\sigma\left(\texttt{syn:ListBodyRest}\right)\\ =&\sigma\left(\texttt{lex:Comma}\;\texttt{syn:ListBodyElement}\;\texttt{syn:ListBodyRest}\right)\\ =&[\sigma\left(\texttt{syn:ListBodyElement}\right)] \circ \sigma\left(\texttt{syn:ListBodyRest}\right) \end{aligned}\] and \[\begin{aligned} \sigma\left(\texttt{syn:ListBodyRest} \right) &= \sigma\left(\texttt{lex:Comma} \right) = []\\ \sigma\left(\texttt{syn:ListBodyRest} \right) &= \sigma\left(\epsilon \right) = [] \end{aligned}\]

Example

// A list with three numbers 1, 2, and 3.
[ 1, 2, 3 ]

4.3 Map type

The type \(\textit{Map}\) represents a list of entries. Each entry is a pair. The first element of a pair is a value of type \(\textit{Name}\), the second element is a value of type \(\textit{Value}\). A value of type \(\textit{Map}\) is expressed in the language by the sentence \(\texttt{syn:Map}\) (as defined in the syntactical grammar).

The semantical representation is defined as \[\begin{aligned} &\sigma\left(\texttt{syn:Map}\right)\\ =&\sigma\left(\texttt{lex:LeftCurlyBracket}\; \texttt{syn:MapBody}\; \texttt{lex:RightCurlyBracket} \right)\\ =&[ \sigma\left(\texttt{syn:MapBody} \right) ]\\ =&[ \sigma\left(\texttt{syn:MapBodyElement}\; \texttt{syn:MapBodyRest} \right) ]\\ =&[ \sigma\left(\texttt{syn:MapBodyElement} \right) ] \circ \sigma\left(\texttt{syn:MapBodyRest} \right) \end{aligned} \] with \[ \sigma\left(\texttt{syn:MapBodyElement}\right) = \sigma\left(\texttt{syn:Value} \right) \] and \[\begin{aligned} &\sigma\left(\texttt{syn:MapBodyRest}\right)\\ =&\sigma\left(\texttt{lex:Comma}\;\texttt{syn:MapBodyElement}\;\texttt{syn:MapBodyRest}\right)\\ =&[\sigma\left(\texttt{syn:MapBodyElement}\right)] \circ \sigma\left(\texttt{syn:MapBodyRest}\right) \end{aligned}\] and \[\begin{aligned} \sigma\left(\texttt{syn:MapBodyRest}\right) &= \sigma\left(\texttt{lex:Comma}\right) = []\\ \sigma\left(\texttt{syn:MapBodyRest}\right) &= \sigma\left(\epsilon\right) = [] \end{aligned}\] and lastly \[\begin{aligned} &\sigma\left(\texttt{syn:MapBodyElement} \right)\\ =&\sigma\left(\texttt{lex:Name};\texttt{lex:Colon};\texttt{syn:Value} \right)\\ =&\left(\sigma\left( \texttt{lex:Name} \right), \sigma\left(\texttt{syn:Value}\right)\right)\\ \end{aligned}\]

Example

// A map of
// text to 'Hello, World!'
// action to 'Print', and
// fontSize to 12.
{ text : 'Hello World!', action : 'Print', fontSize: 12 }

4.4 Number type

The type \(\textit{Number}\) type represents both two's complement integer numbers as well as IEEE754 floating-point numbers (without not-a-number values of infinity values). The values of type number are expressed in the language by the word \(\texttt{Number}\) (as defined in the syntactical grammar). Note that the Data Definition Language does not impose restrictions on the size of the literals or the values. Implementations, however, may impose restrictions.

The semantical representation is defined as \[\begin{aligned} &\sigma\left(\texttt{syn:Number}\right)\\ =&\sigma\left(\texttt{lex:Number}\right) \end{aligned}\]

4.5 String type

The \(\textit{String}\) type represents UTF-8 strings. \(\textit{String}\) values are expressed in the language by the word \(\texttt{syn:String}\)> (as defined in the syntactical grammar). At the end of the syntactical translation of a syn:String word, its escape sequences are replaced by the Unicode symbols they are representing. Furthermore, the opening and closing quotes are removed. Note that the Data Definition Language does not impose restrictions on the size of the literals or the values. Implementations, however, may impose restrictions.

The semantical representation is defined as

\[\begin{aligned} &\sigma\left(\texttt{syn:String}\right)\\ =&\sigma\left(\texttt{lex:String}\right) \end{aligned} \]

4.6 Value type

The type \(\textit{Value}\) type is the union type of the types \(\textit{Boolean}\), \(\textit{List}\), \(\textit{Map}\), \(\textit{Number}\), \(\textit{String}\), and \(\textit{Void}\).

The semantical representation is defined as \[\begin{aligned} &\sigma\left(\texttt{syn:Value}\right)=\sigma\left(\texttt{syn:Boolean} \right)\\ &\sigma\left(\texttt{syn:Value}\right)=\sigma\left(\texttt{syn:List} \right)\\ &\sigma\left(\texttt{syn:Value}\right)=\sigma\left(\texttt{syn:Map} \right)\\ &\sigma\left(\texttt{syn:Value}\right)=\sigma\left(\texttt{syn:String} \right)\\ &\sigma\left(\texttt{syn:Value}\right)=\sigma\left(\texttt{syn:Number} \right)\\ &\sigma\left(\texttt{syn:Value}\right)=\sigma\left(\texttt{syn:Void} \right)\\ \end{aligned} \]

4.7 Void Type

The type sem:Void type has a single values void which is expressed in the language by the word sym: Void (as defined in the syntactical grammar).

The semantical representation is defined as \[\begin{aligned} &\sigma\left(\texttt{syn:Void}\right)\\ =&\sigma\left(\texttt{lex:Void}\right) \end{aligned} \]

4.8 Name type

The values of type \(\textit{Name}\) represent names as expressed in the language by the sentence \(\texttt{lex:Name}\) (as defined in the syntactical grammar).

The semantical representation is defined as \[\begin{aligned} \sigma\left(\texttt{lex:Name}\right) \end{aligned}\]
